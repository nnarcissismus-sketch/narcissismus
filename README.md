# NARCISSISMUS — сайт

Простой статический сайт (HTML/CSS/JS без фреймворков), чёрно-белый,
без анимаций и текстур — старый олдскульный вайб. Работает на GitHub Pages.

## Структура

```
index.html        главная страница
news.html         новости / анонсы
releases.html     релизы (треки/альбомы) со скачиванием .zip
store.html        магазин CD
info.html         инфо (био) + ссылки
lyrics.html       тексты песен
etc.html          всякое разное
style.css         вся стилистика (чёрно-белая, минимум линий)
script.js         логика отрисовки данных из data/*.json
assets/           логотип, фавикон
images/           фото для новостей и обложки релизов/CD
downloads/        .zip файлы с треками/альбомами
data/
  news.json       новости
  releases.json   релизы
  store.json      CD в продаже
  links.json      ссылки на странице info
  lyrics.json     тексты песен
  etc.json        записи в разделе etc
```

## Как выложить сайт на GitHub Pages

1. Создай новый репозиторий на GitHub, например `narcissismus`.
2. Загрузи туда все файлы из этой папки (через веб-интерфейс "Add file →
   Upload files" или через git).
3. В настройках репозитория: `Settings → Pages → Source: Deploy from a
   branch → branch: main → / (root)`.
4. Через минуту сайт будет доступен по адресу вида
   `https://твой-логин.github.io/narcissismus/`.

## Как добавить новость

Открой `data/news.json`, добавь новый блок в список (сайт сам сортирует
по дате):

```json
{
  "date": "2026-08-01",
  "title": "заголовок новости",
  "text": "текст новости.",
  "image": "images/имя-фото.jpg",
  "new": true
}
```

- `image` можно оставить пустым `""`, если фото нет.
- `new` — ставит плашку "NEW". Не забывай убирать её (`false`) со старых
  новостей, если не хочешь, чтобы висела вечно.
- Не забудь загрузить саму фотографию в папку `images/`.

## Как выпустить релиз (трек/альбом)

Открой `data/releases.json`, добавь блок:

```json
{
  "date": "2026-08-01",
  "title": "название альбома",
  "type": "Full Length",
  "release_date": "August, 2026",
  "duration": "41:05",
  "copies": "50 copies",
  "cover": "images/обложка.jpg",
  "tracks": [
    "Track One",
    "Track Two",
    "Track Three"
  ],
  "download": "downloads/имя-архива.zip",
  "new": true
}
```

- `date` используется только для сортировки (новые сверху), можно просто
  ставить сегодняшнюю дату.
- `release_date`, `duration`, `copies` — строки под названием (например
  "Release date: August, 2026"). Любую из них можно оставить `""`, чтобы
  строка не показывалась.
- `tracks` — нумерованный треклист. Можно оставить `[]`, если не нужен.
- Загрузи сам .zip файл в папку `downloads/`.
- Загрузи обложку в `images/` (она показывается справа от текста).

## Как выставить CD на продажу

Открой `data/store.json`, добавь блок:

```json
{
  "id": "unique-id",
  "title": "название альбома",
  "format": "CD, digipak",
  "price": "10 EUR + shipping",
  "cover": "images/обложка.jpg",
  "status": "instock",
  "note": "33 копии"
}
```

`status` может быть:
- `"instock"` — зелёная надпись "IN STOCK"
- `"low"` — жёлтая надпись "LOW STOCK"
- `"soldout"` — красная надпись "SOLD OUT", кнопка заказа отключается

Чтобы пометить партию закончившейся — просто поменяй `status` на
`"soldout"` у нужного диска, ничего удалять не нужно.

## Как редактировать страницу info (био + ссылки)

Текст био на `info.html` — обычный HTML, открой файл и редактируй текст
прямо внутри блока `<div class="section">` под заголовком "info".

Список ссылок вынесен в данные. Открой `data/links.json` и добавь:

```json
{ "label": "Bandcamp", "url": "https://example.bandcamp.com" }
```

## Как добавить текст песни

Открой `data/lyrics.json`, добавь:

```json
{
  "release": "название альбома или сингла",
  "title": "название трека",
  "text": "строка первая\nстрока вторая\nстрока третья"
}
```

Используй `\n` для перевода строки внутри текста.

## Как опубликовать что-то в разделе "etc"

Формат такой же, как у новостей. Открой `data/etc.json`, добавь:

```json
{
  "date": "2026-08-01",
  "title": "о чём это",
  "text": "текст.",
  "image": "images/фото.jpg"
}
```

## Email для заказов

Открой `script.js`, в самом начале поменяй:

```js
var CONTACT_EMAIL = "narcissismus@example.com";
```

на свой настоящий email. Также поменяй email в `index.html`, `info.html`
и `store.html` (ссылки `mailto:`).

## Важно

Если не хочешь редактировать JSON руками — просто пришли текст новости,
релиза, CD, ссылки, текста песни или записи в etc в чат Claude, и он
подготовит готовый файл.
